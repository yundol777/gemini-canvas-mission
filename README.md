## 📚 3. 학습 앱

### 앱 이름
말해 JS

### 배포 링크
https://gemini.google.com/share/7952a563865d

### 이 앱을 만든 이유

- 최근 자바스크립트를 다시 공부 중인데, 영어 공부 어플인 말해보카처럼 매일 5문제씩 자바스크립트 문제를 풀며 모르는 부분을 채울 수 있는 앱을 만들고 싶었습니다.

### 주요 기능

- 16개의 주제 중에서 5개의 문제를 매번 새롭게 생성하여 문제를 생성합니다.
- 문제를 풀며 정답과 오답을 바로바로 확인 가능하며, 결과 페이지에서 정답 여부를 한눈에 확인 가능합니다.
- 다시 풀기 클릭 시 정답을 맞춰야 다음 문제로 넘어가며 하단에 선택한 지문의 틀린 이유와 정답 이유가 표기됩니다.

### AI 기능

- 특정 16가지 주제를 미리 문제은행 식으로 알려주고, 해당 주제에 맞는 문제를 생성하는데 사용하였습니다. 문제의 핵심을 명확히 하여 문제와 답변 수를 적절히 조절하도록 요청하였습니다.
- 문제를 생성하면서 답변에 관한 피드백도 함께 생성하여 저장합니다.

### 2. 프롬프트 개선 과정

- 처음 결과물의 문제점은 무엇이었나요?
결과값으로 나오는 위도, 경도 값과 실제 three.js 기반으로 생성한 지구본의 위치가 맞지 않았습니다.
- 어떤 식으로 프롬프트를 수정했나요?
답변으로 온 사용 가이드에서 위도와 경도를 계산하는 2번 항목이 잘못되었다고 수정을 요청했지만 원하는 결과가 바로 반영되지 않았습니다. 실제 위도 경도의 좌표를 찍어보고 지구본 이미지가 찍히는 좌표값을 비교해가며 위도는 일치하는데 경도가 특정한 수치만큼 차이가 나는 것을 확인했고, 이러한 위도값과 경도값을 기반으로 컴포넌트에 요청을 보내려고 시도했었습니다. 마지막에는 페어들과 코드에서 문제 부분을 찾아 수정해야 하는 코드를 정확히 요청하였고 정상적으로 수정할 수 있었습니다.
- 몇 번의 반복을 거쳤나요?
해당 오류를 수정하기 위해 대략 4번 정도 반복하였습니다.

### 3. 효과적이었던 프롬프팅 전략

- 어떤 방식으로 요청했을 때 원하는 결과가 나왔나요?
코드에 특정 부분이나 코드를 작성하는 것처럼 세세하게 원하는 부분을 분리하여 요청했을 때 조금 더 생각한 대로 결과가 나왔습니다.
- 반대로 효과가 없었던 방식은 어떤게 있었나요?
단순히 문제 상황만 설명하고 수정해달라고 했을 때 어떤 식으로 수치가 차이가 나는지 이해하지 못해서 원하는 방향으로 수정이 어려웠던 것 같습니다. 그래서 실제 three.js에 찍힌 좌표와 실제 지도상 좌표를 비교해 대략적인 위도와 경도를 알려주어 수정을 유도하였습니다.
